# QA Documentation Samples

Automation is only part of QA. This repo shows the other half: planning what to test, writing test cases a teammate could execute without you in the room, and reporting bugs clearly enough that a developer can reproduce them on the first try.

## Contents

- `test-plans/` — a sample test plan for a fictional login feature
- `test-cases/` — manual test cases in a standard tabular format
- `bug-reports/` — sample bug reports showing clear repro steps, expected vs. actual behavior, and severity/priority reasoning

## Why include this

A lot of automation portfolios skip this entirely, which makes it look like the person can script but can't think through *what* to test or communicate a defect clearly — both core QA skills that automation doesn't replace.
